Zarys zadania

Celem zadania jest implementacja hierarchii klas, obrazujących strukturę obiektów w grze strategicznej (na przykładzie Starcraft II) - rasy, budynków i jednostek; oraz narzędzi wspomagajacych.

Hierarchia klas wygląda następująco:

  • Race – klasa zawierająca nazwę rasy Name typu string, oraz konstruktor w postaci Race(string name)
  • Building – klasa dziedzicząca po klasie Race zawierająca nazwę budynku Name typu string, metodę GetRaceName() zwracającą nazwę rasy, oraz konstruktor Building(string Name, string _Race) który ustawia nazwę budynku i nazwę rasy
  • Unit – klasa dziedzicząca po klasie Building zawierająca nazwę jednostki Name typu string, metodę GetBuildingName() zwracającą nazwę budynku, oraz konstruktor Unit(string Name, string Building, string _Race) który ustawia nazwę jednostki, budynku i nazwę rasy
  • L6 – Klasa pomocnicza

Etapy

Etap 1 (2 punkty)

Stwórz hierarchię klas, tak by spełniały założenia z zarysu zadania. Wszystkie klasy powinny przedefinowywać metodę GetHashcode() oraz Equals(object obj)

Etap 2 (1 punkt)

Stwórz klasę L6 - jej składowe to

  • pole _Race typu Race zawierające aktualną rasę oraz metodę SetRace(Race race) która ustawia rasę, nie zmieniając jej jeśli obiekt nie jest typu Race (czyli nie typu Building lub Unit)
  • pole Buildings typu List<Building> zawierające aktualną listę budynków
  • pole Units typu List<Unit> zawierające aktualną listę jednostek
  • AddBuilding, metoda która przyjmuje jeden argument typu Building dodaje go do listy Buildings. Metoda zwraca true jeśli budynek dodano, false jeśli budynek znajduje sie już na liście, null jeśli budynku nie można dodać (element nie jest budynkiem, nie jest odpowiedniej rasy).
  • AddBuildingIgnoreConstraints, metoda która przyjmuje jeden argument typu Building dodaje go do listy Buildings. Metoda zwraca true jeśli budynek dodano, false jeśli budynek znajduje sie już na liście.

Etap 3 (2 punkty)

Rozszerz klasę L6

  • AddUnit, metoda która przyjmuje jeden argument typu Unit dodaje go do listy Units. Metoda zwraca true jeśli jednostkę dodano, false jeśli jednostka znajduje sie już na liście, null jeśli jednostki nie można dodać (element nie jest jednostką, nie jest odpowiedniej rasy, na liście budynków nie ma odpowiedniego budynku).
  • AddUnitIgnoreConstraints, metoda która przyjmuje jeden argument typu Unit dodaje go do listy Units. Metoda zwraca true jeśli jednostkę dodano, false jeśli jednostka znajduje sie już na liście.
  • AreUnitsSameRace - metoda która zwraca true jeśli wszystkie jednostki na liście mają tę samą rasę, false jeśli nie, null jeśli lista jednostek jest pusta.
